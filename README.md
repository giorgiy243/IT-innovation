# IT-innovation

Внутренняя модульная веб-платформа компании: каждый сотрудник под своей ролью получает доступ
к данным и инструментам автоматизации по своему бизнес-домену (продажи, presale, маркетинг,
торги, логистика, документооборот; со временем - новые роли). В перспективе - коммерческий продукт.

Это репозиторий **кода платформы**. Стратегический «мозг» (видение, домены, роли, методология,
архитектурные решения) - в отдельном репозитории `AI/`.

> ⚠️ В репозитории **нет** данных, БД и секретов. `.env`, `*.db`, `data/`, выгрузки `*.xls/*.xlsx`
> исключены в `.gitignore`. Это код - данные подключаются локально, на внутреннем сервере.

## Архитектура (кратко)

Тонкое **ядро** (вход, RBAC, реестр модулей, мастер-данные, аудит, `tenant_id` везде) +
**домены-модули**, которые подключаются по одному. Эталон стиля модуля - `client-rotate`
(FastAPI + self-contained страница).

Подробно: см. `AI/platform/` - `архитектура.md`, `модель_данных.md`, `роли_и_доступ.md`,
`самодостаточность.md`, `решения.md`.

## Стек

Python 3.11+ · FastAPI · PostgreSQL (SQLite - только прототип) · Alembic · Jinja2 · uvicorn.

## Структура

```
core/          ядро: auth, rbac, master_data, registry, audit, db
modules/       домены-модули (пусто на старте; sales - в Фазе 1)
migrations/    Alembic
deploy/        systemd-юнит + скрипт деплоя
tests/
```

## Установка (черновик, наполняется в Фазе 1)

```bash
python -m venv .venv
.venv/Scripts/pip install -r requirements.txt   # Windows
cp .env.example .env                             # заполнить доступ к БД
```

## Ветки и деплой

- `main` - production (то, чем пользуются сотрудники; прямых коммитов нет).
- `staging` - предпросмотр перед слиянием.
- `feature/*` - разработка.
- Деплой из `main`: `git pull -> alembic upgrade -> рестарт` (systemd+uvicorn). Релизы - теги `vX.Y`.
